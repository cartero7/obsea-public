# OBSEA3 Development Workflow

## 📁 Directory Structure

OBSEA3 uses **two directory locations**:

```
~/obsea/                    ← Development workspace (git repo)
├── bin/                    ← Scripts (local development)
├── ros2_ws/src/            ← Source code (edit here)
├── webtest/                ← Web frontend source
└── docs/                   ← Documentation

/opt/obsea/                 ← Production deployment (system-wide)
├── bin/.venv               ← Runtime configuration (copied from ~/obsea/bin/.venv)
├── bin/secrets             ← Security tokens (copied from ~/obsea/bin/secrets)
├── ros2_ws/                ← Built workspace (synced from ~/obsea/ros2_ws)
│   ├── src/                ← Source code (copied from ~/obsea/ros2_ws/src)
│   ├── build/              ← Build artifacts (generated by colcon)
│   └── install/            ← Installed packages (generated by colcon)
├── venv/ros2/              ← Python virtual environment
├── data/                   ← Databases, state files
├── log/                    ← Log files
└── webtest/                ← Web frontend (deployed)
```

## 🔄 Development Workflow

### Phase 1: Initial Setup (Once)

```bash
# Clone repository to your home directory
cd ~
git clone https://github.com/cartero7/obsea.git
cd obsea

# Run automated installer
bash bin/00_install_all.sh
```

This creates:
- Development workspace in `~/obsea/`
- Production deployment in `/opt/obsea/`
- Configuration files in both locations

---

### Phase 2: Daily Development

**Work in your home directory (`~/obsea/`):**

```bash
cd ~/obsea

# 1. Edit source code
vim ros2_ws/src/obsea3_hw_interfaces/obsea3_hw_interfaces/hw_supervisor_node.py
vim ros2_ws/src/obsea3_api_gateway/obsea3_api_gateway/main.py
vim webtest/src/components/Dashboard.jsx

# 2. Sync changes to /opt/obsea and rebuild
bash bin/30_build_ros2_ws.sh

# 3. Restart services to apply changes
bash bin/50_run_all.sh
```

**What happens when you run `30_build_ros2_ws.sh`:**

1. **Syncs source code**: `~/obsea/ros2_ws/src/` → `/opt/obsea/ros2_ws/src/`
2. **Runs colcon build** in `/opt/obsea/ros2_ws/`
3. **Generates build artifacts** in `/opt/obsea/ros2_ws/build/` and `install/`

**Important**: Build ALWAYS happens in `/opt/obsea/ros2_ws/`, never in `~/obsea/ros2_ws/`

---

### Phase 3: Testing Changes

```bash
# Quick smoke tests
bash bin/quick_test.sh

# Full validation
bash bin/comprehensive_test.sh

# Check running services
bash bin/show_config.sh

# View logs
tail -f /opt/obsea/log/current/hw_supervisor.log
tail -f /opt/obsea/log/current/api_gateway.log
```

---

### Phase 4: Git Workflow

```bash
cd ~/obsea

# Check status
git status

# Stage changes (ONLY source code, never build artifacts)
git add ros2_ws/src/
git add bin/
git add docs/

# Commit
git commit -m "feat: your feature description"

# Push to GitHub
git push origin main
```

**Never commit**:
- `ros2_ws/build/`
- `ros2_ws/install/`
- `ros2_ws/log/`
- `bin/.venv` (this is user-specific)
- `bin/secrets` (contains sensitive tokens)
- `webtest/node_modules/`

These are already in `.gitignore`.

## 🔧 Configuration Changes

### Modifying System Configuration

Configuration lives in TWO places:

```bash
~/obsea/bin/.venv          # Local template (edit this during development)
/opt/obsea/bin/.venv       # Active runtime config (used by running services)
```

**To change configuration:**

```bash
# Option 1: Regenerate from scratch
cd ~/obsea
bash bin/00_interactive_setup.sh
sudo bash bin/02_host_setup.sh  # Copies to /opt/obsea

# Option 2: Edit directly (advanced users)
vim ~/obsea/bin/.venv
sudo cp ~/obsea/bin/.venv /opt/obsea/bin/.venv
```

**Then restart services:**
```bash
bash bin/50_run_all.sh
```

### Modifying Runtime Configuration (limits, instruments, etc.)

These files are in `/opt/obsea/ros2_ws/src/obsea3_core/config/`:

```bash
sudo vim /opt/obsea/ros2_ws/src/obsea3_core/config/limits.yaml
sudo vim /opt/obsea/ros2_ws/src/obsea3_core/config/instruments.yaml
```

**Hot-reload supported** for `limits.yaml` (no restart needed).  
Other configs require service restart.

## 🚀 Common Tasks

### Add a new ROS2 node

```bash
cd ~/obsea

# 1. Create Python file
vim ros2_ws/src/obsea3_core/obsea3_core/my_new_node.py

# 2. Register in setup.py
vim ros2_ws/src/obsea3_core/setup.py
# Add to entry_points: 'my_new_node = obsea3_core.my_new_node:main'

# 3. Build and test
bash bin/30_build_ros2_ws.sh
source /opt/obsea/ros2_ws/install/setup.bash
ros2 run obsea3_core my_new_node
```

### Add a new API endpoint

```bash
cd ~/obsea

# 1. Edit FastAPI app
vim ros2_ws/src/obsea3_api_gateway/obsea3_api_gateway/main.py

# 2. Rebuild (Python doesn't need rebuild, but good practice)
bash bin/30_build_ros2_ws.sh

# 3. Restart API
pkill -f uvicorn
bash bin/50_run_all.sh
```

### Update web frontend

```bash
cd ~/obsea

# 1. Edit React components
vim webtest/src/components/Dashboard.jsx

# 2. Development mode (hot reload)
cd /opt/obsea/webtest
npm run dev

# 3. Production build (optional)
npm run build
```

## 📦 Deployment to Production

### Same Machine (Already Deployed)

Your development workflow already deploys to `/opt/obsea/` automatically via `30_build_ros2_ws.sh`.

### Different Machine (Production Server)

```bash
# On development machine
cd ~/obsea
git push origin main

# On production server
git clone https://github.com/cartero7/obsea.git
cd obsea
bash bin/00_install_all.sh
```

Or use `rsync` for direct sync:

```bash
# From development machine to production
rsync -avz --exclude 'build/' --exclude 'install/' \
  ~/obsea/ pi@obsea3-01:/home/pi/obsea/

# On production machine
ssh pi@obsea3-01
cd /home/pi/obsea
bash bin/30_build_ros2_ws.sh
bash bin/50_run_all.sh
```

## 🔍 Troubleshooting Workflow

### "My changes don't appear"

```bash
# 1. Verify source was synced
diff ~/obsea/ros2_ws/src/package/file.py /opt/obsea/ros2_ws/src/package/file.py

# 2. Rebuild
bash bin/30_build_ros2_ws.sh

# 3. Restart services
pkill -f hw_supervisor
pkill -f uvicorn
bash bin/50_run_all.sh
```

### "Permission denied" when building

```bash
# Fix ownership
sudo chown -R $USER:obsea_admins /opt/obsea/ros2_ws
sudo chmod -R g+w /opt/obsea/ros2_ws
```

### "Cannot import module"

```bash
# Ensure virtual environment is active and sourced
source /opt/obsea/venv/ros2/bin/activate
source /opt/obsea/ros2_ws/install/setup.bash

# Check Python path
python3 -c "import sys; print('\n'.join(sys.path))"
```

## 📝 Summary

| Action | Location | Command |
|--------|----------|---------|
| Edit source code | `~/obsea/ros2_ws/src/` | `vim ~/obsea/ros2_ws/src/...` |
| Build workspace | `/opt/obsea/ros2_ws/` | `bash bin/30_build_ros2_ws.sh` |
| Run services | `/opt/obsea/` | `bash bin/50_run_all.sh` |
| View logs | `/opt/obsea/log/current/` | `tail -f /opt/obsea/log/current/*.log` |
| Edit config | `~/obsea/bin/.venv` → `/opt/obsea/bin/.venv` | `vim ~/obsea/bin/.venv` + copy |
| Git operations | `~/obsea/` | `git add/commit/push` |

**Remember**: 
- ✅ **Develop** in `~/obsea/`
- ✅ **Build** in `/opt/obsea/`
- ✅ **Run** from `/opt/obsea/`
- ✅ **Commit** from `~/obsea/`
